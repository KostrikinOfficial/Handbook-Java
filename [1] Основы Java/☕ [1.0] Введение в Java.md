# ☕ [1.0] Введение в Java
___
## ☕ [1.0.1] Java - это
### Объектно-ориентированный язык программирования общего назначения, созданный с упором на переносимость, надежность и безопасность.
### Java платформонезависимый язык, поскольку при помощи JVM программа может запускаться на любой платформе, будь-то Windows, Linux, macOS и остальные, при условии что для данной операционной системе есть версия JVM
## ☕ [1.0.2] JDK, JRE, JVM
### 🛠️ JDK - это набор инструментов для разработки Java-проекта. Содержит в себе компилятор javac, JRE и стандартные библиотеки. Нужен для разработки программы
###
### ▶️ JRE - это среда выполнения Java. Содержит в себе JVM и стандартные библиотеки. Нужен для запуска программы
###
### ⚙️ JVM - это виртуальная машина, выполняющая байткод. Содержит в себе:
#### 📦 Class Loader (Загружает новые классы)
#### ▶️ Интерпретатор (Выполняет байткод инструкцию за инструкцией)
#### ⚡ JIT Compiler (Компилирует часто выполняемый код в машинный)
#### 🗑️ Garbage Collector (Освобождает память от ненужных объектов)
___
## ☕ [1.0.3] Схема выполнения программы Java
###
    ☕ Main.java (Java код)
            |
            |
            ▼
    🛠️ javac (Компилятор)
            |
            |
            ▼
    📄 Main.class (Байткод)
            |
            |
            ▼
    ⚙️ JVM (начинает выполнение)
            |
            |_ 📦 Class Loader
            |
            |_ ▶️ Интерпретатор
            |
            |_ ⚡ JIT Compiler
            |
            |_️ 🗑️ Garbage Collector
            |
            |
            ▼
    💻 Выполнение (Машинный код выполняется процессором)

###
#### ☕ Main.java - программа, написанная на Java. Но компьютер его не понимает.
###
#### 🛠️ Компилятор javac проверяет синтаксис, ищет ошибки и преобразует Java-код в байткод. Получается файл: `Main.class`
###
#### 📄 Байткод - промежуточный код между Java-кодом и машинным.
#### ❗ Процессор не понимает байткод, его понимает JVM, поэтому файл .class можно запустить на любых платформах.
###
#### ⚙ JVM начинает выполнение с главного метода `public static void main(String[] args)`
#### 📦 Class Loader - когда JVM впервые требуется какой-то класс, будь-то `Scanner` или `String`, Class Loader загруждает его в память.
#### ▶️ Интерпретатор - JVM выполняет байткод инструкцию за инструкцией
#### ⚡ JIT Compiler наблюдает за программой. Если JIT замечает, что участок программы выполняется слишком часто, он понимает: _Этот участок выполняется много раз!_ Тогда он переведит именно этот участок в машинный код и сохраняет его в памяти. Теперь JVM больше не интерпретирует этот участок каждый раз. Она сразу выполняет готовый машинный код. Это значительно ускоряет программму.
#### 🗑️ Garbage Collector - допустим есть класс `Student student = new Student();`. Позже этот объект стал не нужен. Его на него больше никто не ссылается, Gardbage Collector находит такой объект и освобождает память.
###
####  💻 Выполнение процессором - в итоге процессор выполняет уже машинный код. Именно благодаря этому программа начинает реально работать.
